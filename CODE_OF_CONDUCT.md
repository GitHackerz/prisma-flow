# Code of Conduct

## Our Pledge

We want PrismaFlow to be a practical, respectful, and welcoming project for developers working on Prisma database safety.

Participants are expected to:

- Be respectful and constructive.
- Assume good intent, but take responsibility for impact.
- Keep discussions focused on the project and user outcomes.
- Accept maintainers' decisions on scope, roadmap, and review standards.

## Unacceptable Behavior

Unacceptable behavior includes harassment, personal attacks, discriminatory language, sustained disruption, publishing private information, or pressuring maintainers or contributors outside the project.

## Enforcement

Maintainers may remove comments, close issues, reject contributions, or block participants who violate this code of conduct.

If you need to report a concern, contact the maintainers privately. Include links, screenshots, or context when available.
